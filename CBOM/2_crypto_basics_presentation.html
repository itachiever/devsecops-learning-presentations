What is Cryptography?

📖 Cryptography = the science of writing and reading secret messages. In computing, it means converting readable data into a scrambled form so that only authorised parties can access it — keeping data confidential, intact, and authentic.

🌍 Why Do We Need It?
  • 🔓
    Without it, everything you type online — passwords, card numbers — travels as plain readable text that anyone can intercept.
  • 🔒
    Cryptography ensures only the intended recipient can understand the data, even if it is captured mid-way.
  • ✅
    It also verifies identity — confirming that a website or person is genuinely who they claim to be.
📌 Real-World Examples
🌐

HTTPS websites — the padlock in your browser means your connection is encrypted using TLS cryptography.

🔑

Login & passwords — your password is never stored as-is; cryptography converts it into an unreadable hash.

💬

Secure messaging — apps like WhatsApp use end-to-end encryption so only sender & receiver can read messages.

💡 Analogy: Imagine passing a secret note in class, but written in a code only you and your friend know. Anyone else who grabs it sees gibberish. That code is cryptography.

Encryption Explained

📖 Encryption is the process of converting readable data (plaintext) into an unreadable scrambled form (ciphertext) using a mathematical key. Only someone with the correct key can reverse it back to plaintext.

🔐 Data Protection Examples
  • 🌐
    HTTPS / TLS — every website visit over https:// uses encryption to protect your data in transit.
  • 💳
    Online payments — card details are encrypted before leaving your browser, reaching the bank safely.
  • 🗄️
    Database encryption — sensitive records stored on servers are encrypted so even a stolen database is useless.
🗝️ Key Terms to Remember
  • 📄
    Plaintext — the original, readable message before encryption.
  • 🔒
    Ciphertext — the scrambled, unreadable output after encryption.
  • 🗝️
    Key — a secret value used to encrypt and/or decrypt data.
  • ↩️
    Decryption — reversing ciphertext back to plaintext using the correct key.
🧮 Core Concept · Slide 4 of 6

Hashing Explained

📖 Hashing converts any data into a fixed-size fingerprint (called a hash or digest). It is a one-way function — you can generate the hash, but you cannot reverse it back to the original data. It proves integrity, not secrecy.

⚙️ How Hashing Works

Input: Your Password

"MyPassword123"

⬇️ SHA-256 algorithm

Hash Output (stored in DB)

"ef92b778bafe771207…"

💡 Same input always produces the same hash. Even a tiny change — one character — completely changes the output.

🔐 Real-World Usage
  • 🔑
    Password storage — websites hash your password, store only the hash. At login, they hash what you type and compare — the real password is never stored.
  • 📦
    File integrity — software downloads publish a hash. You hash the file yourself and compare — if they match, the file is untampered.
  • ⛓️
    Blockchain — every block contains the hash of the previous block, creating a tamper-proof chain.
⚡ Hashing vs Encryption — Key Difference
DirectionEncryption: two-way (encrypt & decrypt)  |  Hashing: one-way only
PurposeEncryption: secrecy in transit  |  Hashing: integrity & verification
Key?Encryption: needs a key  |  Hashing: no key needed
🔑 Types · Slide 5 of 6

Types of Encryption

There are two fundamental types of encryption. Symmetric uses one shared secret key for both locking and unlocking. Asymmetric uses a key pair — a public key to lock and a private key to unlock. Both are essential in modern security.

🔵 Symmetric Encryption
How it worksOne single key is used to both encrypt and decrypt the data.
AnalogyA lockbox where the same physical key locks and unlocks it.
AlgorithmAES-256 — the gold standard used in HTTPS, VPNs, disk encryption.
SpeedVery fast — ideal for encrypting large amounts of data.
RiskBoth parties must share the key securely — if the key leaks, all data is exposed.
Used inFile encryption, database encryption, VPN tunnels, Wi-Fi (WPA2/3).
🟣 Asymmetric Encryption
How it worksUses a key pair: a public key encrypts, a private key decrypts.
AnalogyA mailbox with a slot anyone can drop letters into (public), but only you hold the key to open it (private).
AlgorithmRSA-2048 / 4096 — used in HTTPS certificates, SSH, digital signatures.
SpeedSlower — best for small data like key exchange and identity verification.
AdvantagePublic key can be freely shared — no risk in distributing it to anyone.
Used inHTTPS (TLS handshake), SSH login, digital certificates, email signing.

💡 Real-world combo: HTTPS first uses asymmetric encryption (RSA/ECDH) to securely exchange a temporary key, then switches to symmetric encryption (AES) for the actual data transfer — getting the best of both worlds.

🏁 Recap · Slide 6 of 6

Summary & Key Takeaways

🔒

Cryptography

The science of protecting information by transforming it so only authorised parties can read it. Powers all digital security.

🔄

Encryption

Converts plaintext → ciphertext using a key. Two-way — data can be decrypted back. Protects data in transit and at rest.

🧮

Hashing

Creates a unique fixed fingerprint. One-way — cannot be reversed. Used for password storage and file integrity checks.

🔵

Symmetric

One shared key for encrypt & decrypt. Fast — great for large data. Example: AES-256 used in VPNs and storage.

🟣

Asymmetric

Public key encrypts, private key decrypts. Enables secure key exchange without sharing secrets. Example: RSA in HTTPS.

🛡️

Why It Matters

Without cryptography: passwords, banking, and all private communications would be completely exposed to attackers.
