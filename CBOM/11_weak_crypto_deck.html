Weak cryptography is the most common and preventable security risk in modern software. Legacy algorithms still run silently inside millions of applications. CBOM makes these risks visible — so they can be detected, prioritised, and eliminated before attackers exploit them.

💀 Definition · Slide 2 of 6

What is Weak Cryptography?

📖 Weak cryptography refers to cryptographic algorithms, protocols, or key configurations that are mathematically broken, computationally crackable, or officially deprecated by standards bodies (NIST, IETF, ENISA). They may have worked when first designed but are no longer safe against modern attacks — yet they persist in production systems worldwide.

💀 MD5 BROKEN — Never Use
Message Digest 5 — designed in 1991. Collision attacks proven in 2004. Two completely different inputs can produce the same MD5 hash, making signatures and integrity checks worthless.
Still found in: legacy password storage · file checksums · token generation in old APIs
1991 / Broken 2004
⚡ SHA-1 BROKEN — Migrate Now
Secure Hash Algorithm 1 — Google's SHAttered attack (2017) generated two PDFs with identical SHA-1 hashes for only $110,000. Certificate authorities stopped issuing SHA-1 certificates in 2016.
Still found in: legacy TLS certs · old Git objects · software signing in legacy pipelines
1995 / Broken 2017
🔌 TLS 1.0 / 1.1 DEPRECATED — Disable
TLS 1.0 (1999) and TLS 1.1 (2006) use weak cipher suites and have known attack vectors (POODLE, BEAST). RFC 8996 officially deprecated both in March 2021. Major browsers removed support.
Still found in: legacy APIs · IoT devices · old load balancers · enterprise middleware
Deprecated March 2021
🗝️ DES / 3DES RETIRED — Remove
DES (56-bit key) was cracked in 22 hours in 1999. 3DES (Triple-DES) was NIST-deprecated in 2023 — vulnerable to SWEET32 birthday attack and too slow for modern applications.
Still found in: banking mainframes · legacy payment systems · old VPN configs · PKCS#12 files
DES: Broken 1999 / 3DES: Deprecated 2023

⚠️ Short Keys: RSA-512 can be factored in hours on commodity hardware. RSA-1024 in weeks. NIST minimum: RSA-2048 until 2030, then RSA-3072+ required. CBOM flags any key below minimum safe length.

⚠️ Weak Modes: Even strong algorithms become weak with bad configuration. AES in ECB mode reveals data patterns visually. CBOM tracks algorithm mode, not just algorithm name.

🚨 Risk Landscape · Slide 3 of 6

Risks of Weak Cryptography — Real-World Impact

📖 Weak cryptography is not a theoretical risk — it is actively exploited every day. When an organisation unknowingly uses broken algorithms, attackers have a proven, repeatable method to compromise data, impersonate services, and breach systems — often without leaving any trace.

💥

Data Breaches

🔴 Critical
If a database of MD5-hashed passwords is stolen, attackers use rainbow table lookups to reverse millions of hashes in seconds. No brute-force needed — it's a simple lookup in a pre-computed table of known hash-to-password mappings.
Real incident: 2012 LinkedIn breach — 117M MD5 passwords cracked within days of the dump being published online.
🕵️

Man-in-the-Middle Attacks

🔴 Critical
TLS 1.0 is vulnerable to the POODLE attack — an attacker in the network path can force a protocol downgrade and then decrypt the session. All data transmitted — passwords, session tokens, card numbers — becomes readable.
Real impact: Public Wi-Fi + TLS 1.0 endpoint = attacker on same network can read all traffic with freely available tools.
🔓

Password & Signature Forgery

🟠 High
SHA-1 collision vulnerabilities allow attackers to create a forged document or certificate with the same hash as a legitimate one. A software update signed with SHA-1 could be replaced with malware — and verification passes.
Real incident: SHAttered (2017) — first practical SHA-1 collision creating two different PDFs with identical signatures.
📋

Compliance Failures & Fines

🟠 High
PCI-DSS 4.0, HIPAA, FIPS 140-3, and GDPR all mandate specific minimum cryptographic standards. Using MD5 for any security function or TLS 1.0 for cardholder data transmission is an automatic compliance violation — even if no breach has occurred.
Real impact: PCI-DSS audit failure for TLS 1.0 usage can result in suspension of card processing and fines up to $100K/month.

⚛️ The Quantum Threat — The Next Wave: Quantum computers will break RSA-2048 and ECC-256 in minutes using Shor's algorithm. Organisations need to map their current crypto now using CBOM to plan migration to post-quantum algorithms (CRYSTALS-Kyber, CRYSTALS-Dilithium) before quantum hardware becomes accessible.

🔧 Remediation · Slide 5 of 6

Risk Management & Remediation Playbook

📖 Detection without action is meaningless. CBOM provides the visibility layer; the remediation playbook provides the action layer. For each category of weak cryptography, there is a clear, proven migration path — and CBOM tracks each one to completion.

💀

Replace Broken Hash Algorithms

Priority: Critical
MD5→SHA-256 / SHA-3
SHA-1→SHA-256 / SHA-512
MD5 passwords→bcrypt / Argon2id
CBOM tracks: old algorithm removed ✓ · new algorithm verified ✓ · all call sites updated ✓
🔌

Upgrade TLS Versions & Ciphers

Priority: High
TLS 1.0 / 1.1→TLS 1.3 preferred
AES-CBC suites→AES-256-GCM suites
RSA key exchange→ECDHE (forward secrecy)
CBOM verifies: new protocol active in runtime scan · old version disabled · cipher suite policy enforced
🗝️

Key Rotation & Strengthening

Priority: Medium
RSA-1024→RSA-2048 / RSA-4096
Never-rotated keys→90-day rotation policy
Hardcoded keys→HSM / KMS managed
CBOM tracks: key age · last rotation date · storage location · rotation policy compliance
📜

Certificate Lifecycle Management

Ongoing
SHA-1 signed certs→SHA-256 reissued certs
Manual renewal→cert-manager auto-renew
Expired cert in prod→30-day expiry alert
CBOM continuously monitors: expiry countdown · issuer validity · chain integrity · OCSP revocation status
