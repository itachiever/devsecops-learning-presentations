Overview of the CBOM Report

📖 A CBOM (Cryptographic Bill of Materials) is a structured inventory of every cryptographic component inside a software project. This report was auto-generated by scanning the Keycloak source code — it contains every algorithm, key type, and protocol Keycloak uses, along with where each one appears in the code.

🏁 Summary & Key Insights · Slide 6 of 6

What We Learned from This CBOM Report

⚠️

MD5 Still in Use

Found in ComputedKey.java line 34. MD5 is cryptographically broken since 2004. Should be replaced with SHA-256 immediately.

🔑

SHA1withRSA & SHA1withDSA

Deprecated SHA-1 based signatures found in SAML SignatureUtil. SHA-1 collisions proven since 2017 — must migrate to SHA256withRSA minimum.

🔌

TLS Version Unknown

TLS detected in 15 files but the version (1.2 or 1.3) is not captured by static scanning alone. Runtime scanning needed to verify TLS 1.2+ is enforced.

✅

Strong Algorithms Present

SHA-256, AES128-GCM, HMAC-SHA256, ConcatenationKDF — all good modern symmetric choices. SHA256withRSA, SHA384withRSA, SHA512withRSA also present for RSA signing.

⚛️

Quantum Risk Flagged

8+ algorithms flagged "Not Quantum Safe": RSA-2048, EC, ECDH, DSA, EdDSA, RSASSA-PSS, SHA256withRSA family. Post-quantum migration planning should begin now.

Full Code Traceability

Every algorithm traced to exact file, line, and method. DevSecOps teams can immediately find, assign, and fix any finding — no investigation lag.
