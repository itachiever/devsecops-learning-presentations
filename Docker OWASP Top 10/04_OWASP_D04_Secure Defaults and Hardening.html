D04 – Secure Defaults and Hardening
Building Secure Docker Containers from the Ground Up
🔐 Essential Security for Every Docker Team
OWASP
2/10

🔧 What are Secure Defaults?

  • ✅ Pre-configured settings that prioritize security over convenience
  • 🛡️ "Safe by default" approach - systems start secure, not easy
  • 🔒 Minimal permissions, disabled unnecessary features, restricted access
  • ⚡ Prevents common security mistakes from day one
Example: SSH disabled by default, not running on port 22
3/10

🔨 What is Hardening?

  • 🛠️ Process of securing a system by reducing attack surface
  • ❌ Removing unnecessary services, features, and access points
  • 🔐 Applying additional security configurations beyond defaults
  • 📋 Following security benchmarks and best practices
Example: Removing package managers, shell access, and debug tools from production containers
4/10

⚠️ Why Defaults Matter

🚨 Risks of "Easy but Unsafe"

  • 🏃‍♂️ Teams rush to production without security review
  • 🔓 Default credentials and open permissions
  • 📡 Unnecessary services exposed to networks
  • 🎯 Easy targets for automated attacks
💡 80% of breaches exploit weak defaults and misconfigurations!
5/10

🛡️ Security Configuration Examples

❌ Disable:

  • SSH daemon in containers
  • Debug and development tools
  • Unnecessary network services

✅ Enable:

  • Read-only root filesystem
  • Non-root user execution
  • Resource limits and quotas
docker run --read-only --user 1000:1000 --memory=512m myapp
6/10

⚔️ Attack Vectors

  • 🌐 Exposed APIs: Management interfaces left open to internet
  • 👑 Privilege Escalation: Containers running as root unnecessarily
  • 🚪 Backdoors: Debug tools and SSH access in production
  • 📊 Information Disclosure: Verbose error messages and logs
Real Attack: Docker API on port 2376 exposed → Full host compromise
7/10

💥 Real-World Impact

  • 🏢 Tesla (2018): Exposed Kubernetes dashboard led to cryptocurrency mining
  • 💻 Docker Hub breach: Default credentials on admin accounts
  • 🏦 Financial services: $2M loss from misconfigured containers
⏱️ Average breach detection: 197 days
💰 Average cost: $4.45M
8/10

🔍 How to Identify Weak Defaults

  • 📊 Benchmarks: CIS Docker Benchmark, NIST guidelines
  • 🛠️ Tools: Docker Bench Security, Trivy, Clair scanners
  • 📋 Audits: Regular security assessments and penetration testing
  • 🔎 Monitoring: Runtime security monitoring for anomalies
docker run --rm -it --net host --pid host --userns host --cap-add audit_control \
  -v /var/lib:/var/lib:ro docker/docker-bench-security
9/10

🛡️ Prevention Steps

🔧 Configuration:

  • Use minimal base images (Alpine, Distroless)
  • Run as non-root user
  • Set resource limits

📋 Process:

  • Security-first Dockerfile templates
  • Automated security scanning in CI/CD
  • Regular benchmark compliance checks
FROM alpine:latest
RUN addgroup -g 1000 appgroup && adduser -D -u 1000 -G appgroup appuser
USER appuser
10/10

🎯 Key Takeaways

  • 🔐 Start Secure: Configure security from day one, not as afterthought
  • 📏 Follow Standards: Use CIS benchmarks and security frameworks
  • 🔄 Automate Checks: Integrate security scanning in your pipeline
  • 👥 Team Training: Educate developers on secure defaults
  • 📊 Monitor & Audit: Regular assessment and continuous improvement
🏆 Remember: Security is everyone's responsibility!
