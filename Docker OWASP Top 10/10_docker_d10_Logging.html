D10 – Logging

Essential Security for Container Environments

2/10

📝 What is Logging?

Simple Definition

  • Recording events and activities in your applications
  • Creating a permanent record of what happened and when
  • Capturing system behavior, errors, and user actions
  • Essential for debugging, monitoring, and security analysis

Think of logs as your application's diary 📔

3/10

🛡️ Why Logs Matter

🔒 Security Benefits

  • Detect security incidents
  • Track unauthorized access
  • Monitor suspicious patterns

⚙️ Operational Benefits

  • Debug application issues
  • Monitor system performance
  • Meet compliance requirements
4/10

📋 What Should Be Logged

Docker & Container Examples:

  • Authentication: Login attempts, failures, privilege escalations
  • Container Events: Start/stop, resource usage, configuration changes
  • Network Activity: Connections, port access, traffic patterns
  • File System: File access, modifications, permission changes
  • Application Events: Errors, transactions, API calls
⚠️ Never Log: Passwords, API keys, personal data, or secrets!
5/10

⚠️ Impact of Missing Logs

🚨 Real Security Risks

  • Blind to attacks: Can't detect ongoing breaches
  • No forensics: Can't investigate what happened
  • Compliance failures: Legal and regulatory violations
  • Extended damage: Attacks go unnoticed for months
  • No accountability: Can't trace who did what

Without logs, you're flying blind! 🙈

6/10

🎯 How Attackers Exploit Weak Logging

Common Attack Vectors:

  • Log tampering: Deleting or modifying evidence of their activities
  • Avoiding detection: Using techniques they know won't be logged
  • Persistent access: Maintaining backdoors without leaving traces
  • Privilege escalation: Escalating without triggering alerts
Real Example: Attacker compromises container, disables logging, then moves laterally through the network undetected for weeks.
7/10

🔍 Identifying Weak Logging

Red Flags to Check:

  • Missing logs: No authentication, error, or access logs
  • Insufficient detail: Logs lack timestamps, user IDs, or context
  • Local only: Logs stored only on the container itself
  • No rotation: Logs fill up disk space and get overwritten
  • Unprotected: Anyone can read, modify, or delete logs
Quick Test: Ask yourself - "If we were breached right now, would our logs tell us how?"
8/10

✅ Logging Best Practices

🛠️ How to Fix It

  • Centralize: Send logs to external systems (ELK, Splunk)
  • Rotate: Automatically archive old logs to prevent disk fills
  • Protect: Secure log files with proper permissions
  • Structure: Use JSON format for easier analysis
  • Time sync: Ensure accurate timestamps across containers
docker run --log-driver=syslog --log-opt syslog-address=tcp://logs.company.com:514
9/10

🔔 Monitor & Act on Logs

🚨 Set Up Alerts For:

  • Failed login attempts
  • Privilege escalations
  • Unusual network activity
  • Container crashes
  • Critical application errors

📊 Regular Analysis:

  • Review logs weekly for patterns
  • Use automated correlation tools
  • Create dashboards for key metrics
  • Train teams on log interpretation
Pro Tip: Logs are only valuable if someone is watching them!
10/10

🎯 Key Takeaways

📝 Remember These Essentials

  • Centralize all logs - Don't store them locally on containers
  • Protect log integrity - Secure permissions and access controls
  • Never log secrets - Keep sensitive data out of logs
  • Set up monitoring - Logs without alerts are just storage
  • Test regularly - Ensure logging works when you need it

Good logging = Better security! 🛡️
