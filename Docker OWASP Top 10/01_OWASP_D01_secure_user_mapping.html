D01 - Secure User Mapping

Docker Security

Learn to run Docker containers securely with proper user permissions

Source: OWASP Docker Security Project

What is Secure User Mapping?

👤
  • Running containers with specific, non-privileged user accounts
  • Mapping container users to host system users safely
  • Avoiding root privileges inside containers whenever possible
  • Essential Docker security practice for production environments
  • Implements principle of least privilege in containerized apps
Source: OWASP Docker Security Project

Why is Secure User Mapping Critical?

⚠️
  • Root containers = full system access if breached
  • Container escape can compromise entire host system
  • Principle of least privilege reduces attack surface significantly
  • Never run containers as root in production!
  • Compliance requirements (SOX, PCI-DSS, HIPAA) mandate this
Source: OWASP Docker Security Project

D01 Attack Vectors - How Attackers Exploit Poor User Mapping

🚨

🔓 Container Breakout

Root containers allow kernel exploits to escape sandbox and access host

📁 File System Access

Root access enables reading/writing sensitive host files and directories

⬆️ Privilege Escalation

Attackers exploit SUID binaries and capabilities to gain root privileges

🔍 Process Hijacking

Root containers can manipulate host processes and system services

How to Implement Secure User Mapping

🔧

Dockerfile Method

FROM alpine RUN adduser -D -s /bin/sh appuser USER appuser WORKDIR /home/appuser

Runtime Method

docker run --user 1001:1001 \ -v /host/data:/data \ myapp:latest
  • Always specify non-root users (UID > 0)
  • Run with least privilege principle
Source: OWASP Docker Security Project

Advanced: User Namespace Remapping

🔄
  • Maps container root to unprivileged host user automatically
  • Provides additional security layer at Docker daemon level
  • Container UID 0 ≠ Host UID 0 (mapped to high UID)
  • Even if container runs as root, host sees non-root
# /etc/docker/daemon.json { "userns-remap": "default" } # Restart Docker daemon sudo systemctl restart docker

Summary & Immediate Action Plan

🔐 Core Principle

Never run containers as root! Always use least privilege principle.

🛠️ Implementation

Use USER directive in Dockerfile or --user flag at runtime

🔄 Advanced Security

Enable user namespace remapping at Docker daemon level

✅ Verification

Always verify with docker inspect and continuous monitoring

# Quick Security Check docker ps --format "table {{.Names}}\t{{.Image}}" for container in $(docker ps -q); do echo "Container: $(docker inspect $container --format '{{.Name}}')" echo "User: $(docker inspect $container --format '{{.Config.User}}')" done
