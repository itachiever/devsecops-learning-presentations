🔑 D06: PROTECT SECRETS

Securing Sensitive Data in Docker Containers

Slide 2 of 10
🔑 What is a "Secret" in Docker?
A secret is any sensitive information your application needs to function
  • Passwords and authentication tokens
  • API keys and certificates
  • Database connection strings
  • Private keys and encryption keys
💡 Example
DATABASE_PASSWORD=super_secret_password123
API_KEY=sk-1234567890abcdef JWT_SECRET=my-jwt-signing-key
Slide 3 of 10
🛡️ Why Secrets Need Protection in Containers
  • Containers are ephemeral and often shared
  • Images can be inspected by anyone with access
  • Container logs may be visible to operators
  • Orchestration platforms store container configs
🎯 Key Point

Unlike traditional servers, containers expose more attack surfaces for secrets. Proper protection is essential from day one.

Slide 4 of 10
💥 What Happens When Secrets Are Exposed?
  • Complete database compromise and data theft
  • Unauthorized API access and resource abuse
  • Identity theft and privilege escalation
  • Financial losses and compliance violations
🚨 Real Impact

A hardcoded AWS key in a Docker image led to $100K+ in unexpected charges when attackers launched cryptocurrency miners in the compromised account.

🔑 Keys & Certificates

Slide 6 of 10
🐛 How Secrets Commonly Get Leaked
  • Hardcoded in Docker images or source code
  • Passed as environment variables in plain text
  • Stored in configuration files within containers
  • Logged accidentally in application outputs
⚠️ Attack Vector Example
# BAD: Secret in Dockerfile ENV DATABASE_PASSWORD=secret123
# BAD: Secret in docker run command docker run -e API_KEY=sk-12345 myapp
Slide 7 of 10
🔍 How to Identify if Secrets Are Exposed
  • Use security scanners like TruffleHog or GitLeaks
  • Inspect Docker images with "docker history"
  • Check environment variables in running containers
  • Review application logs for accidental exposure
🔧 Quick Check Commands
# Inspect image layers docker history myapp:latest
# Check container environment docker exec container_id env
# Scan for secrets trufflehog docker --image myapp:latest

🏦 How to Store and Manage Secrets Safely

☁️ Secret Managers

  • AWS Secrets Manager
  • HashiCorp Vault
  • Azure Key Vault

🐳 Docker Tools

  • Docker Secrets
  • Kubernetes Secrets
  • External secret operators
✅ Best Choice

Use dedicated secret management services that provide encryption, rotation, and audit trails.

Slide 9 of 10
⚙️ Step-by-Step: Injecting Secrets Securely at Runtime
  • Step 1: Store secrets in a secret manager
  • Step 2: Configure container to fetch at startup
  • Step 3: Use temporary files or memory storage
  • Step 4: Clean up secrets when container stops
💻 Docker Swarm Example
# Create secret echo "mypassword" | docker secret create db_password -
# Use in service docker service create --secret db_password myapp:latest
Slide 10 of 10
🎯 Best Practices & Recap
  • Never hardcode secrets in images or code
  • Always use dedicated secret management tools
  • Rotate secrets regularly and automate when possible
  • Monitor for secret exposure with scanning tools
  • Train your team on secure practices
