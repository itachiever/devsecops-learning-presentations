OWASP Docker Top 10

D08: Container Image Integrity and Origin

What is a Container Image? 📦

  • A lightweight, standalone package containing everything needed to run an application
  • Includes code, runtime, system tools, libraries, and settings
  • Built from layers defined in a Dockerfile
  • Stored in registries like Docker Hub, ECR, or private repositories
📋 Example: nginx:latest contains the Nginx web server, configuration files, and all dependencies ready to run

Why Image Integrity Matters 🛡️

  • Container images can contain malware or backdoors
  • Compromised images can lead to data breaches and system takeovers
  • Supply chain attacks target popular base images
  • Modified images may contain vulnerable or outdated components
⚠️ Risk: A single malicious image can compromise your entire container environment and spread to other systems

Trusted Origin Sources ✅

✅ Trusted Sources

  • Official Docker Hub images
  • Verified publisher images
  • Your organization's private registry
  • Signed images with cryptographic verification

❌ Untrusted Sources

  • Random user repositories
  • Unverified third-party images
  • Images without clear maintainer
  • Unsigned or tampered images

D08 Best Practices 🎯

  • Use official base images: python:3.9-slim instead of randomuser/python
  • Verify image signatures using Docker Content Trust
  • Pin specific image versions with SHA256 hashes
  • Regularly update base images to latest secure versions
📋 Good Practice:
FROM python:3.9.16-slim@sha256:abc123...
Uses official image with specific version and hash verification

Common Attack Vectors 🚨

  • Typosquatting: Images with names similar to popular ones
  • Backdoored images: Legitimate images with hidden malicious code
  • Supply chain poisoning: Compromising upstream dependencies
  • Registry takeover: Attackers gaining control of image repositories
🐛 Real Attack: Cryptomining malware found in popular "node.js" images (typo of "nodejs") downloaded 100,000+ times

Impact of Ignoring D08 💥

  • Data breaches: Sensitive information stolen or exposed
  • System compromise: Attackers gain persistent access
  • Service disruption: Malware causing downtime and instability
  • Compliance violations: Regulatory fines and audit failures
💸 Business Impact: Average cost of container security incident: $4.24M including downtime, investigation, and remediation

Identifying Image Problems 🔍

  • Scan images with tools like Trivy, Clair, or Snyk
  • Check image source and maintainer reputation
  • Verify digital signatures and checksums
  • Monitor unusual network activity from containers
🔧 Quick Check:
docker history image:tag - Review image layers
trivy image image:tag - Scan for vulnerabilities

Prevention & Remediation 🛠️

  • Implement image allowlists with approved base images only
  • Enable automated vulnerability scanning in CI/CD pipelines
  • Use private registries with access controls
  • Regularly patch and rebuild images with latest updates
🚀 Prevention Strategy: Set up admission controllers (like OPA Gatekeeper) to automatically reject non-compliant images at deployment time

Key Takeaways & Best Tips 📚

  • Always use official images from trusted publishers
  • Scan everything - automate vulnerability scanning
  • Keep images updated with regular security patches
  • Verify integrity using signatures and checksums
🎯 Golden Rule: Treat container images like any other software - verify source, scan for vulnerabilities, and keep updated. Never trust, always verify!
