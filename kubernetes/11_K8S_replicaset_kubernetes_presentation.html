Definition
What is a ReplicaSet?
🎯
Ensures a specified number of Pods always run — you tell it "I want 3 Pods", it makes sure there are always 3
👁
Monitors Pods continuously — watches every Pod it manages in real-time, 24/7
🔁
Auto-creates replacement Pods — detects a crashed Pod and immediately schedules a new one
⚡
Maintains desired state — the target count you define is always matched, no matter what happens
One sentence: ReplicaSet = an automatic Pod manager that guarantees your app always has the right number of instances running.

🎯Desired State
What you define in the ReplicaSet spec — the number of Pods you want running at all times

Scaling
Scaling with ReplicaSet
📈
Scale up → more Pods created — increase replicas from 3 to 5, RS adds 2 new Pods instantly
📉
Scale down → Pods removed — decrease replicas from 5 to 2, RS terminates 3 Pods gracefully
🚦
Handles traffic spikes — scale up when load increases, scale down to save resources when it drops
↔️
Horizontal Pod Autoscaler (HPA) — can auto-scale ReplicaSet based on CPU/memory metrics
Scale up command
kubectl scale rs myapp --replicas=5
Scale down command
kubectl scale rs myapp --replicas=2

Key Details
Important Things to Know About ReplicaSet
🏷
Uses labels & selectors to identify Pods — RS finds its Pods by matching labels, not by name. Any Pod matching the selector is "owned"
📦
Manages Pods, not containers — RS creates/deletes Pods. The container inside is managed by Kubelet on the node
🏗
Usually managed by Deployments — in real production, you rarely use ReplicaSet directly. Deployments wrap it and add rolling updates
⚠️
Rarely used directly in production — always prefer Deployments over raw ReplicaSets unless you have a specific reason not to
# ReplicaSet YAML example
apiVersion: apps/v1
kind: ReplicaSet
spec:
  replicas: 3 # desired count
  selector:
    matchLabels:
      app: my-app # ← label selector
  template:
    metadata:
      labels:
        app: my-app # must match ↑
