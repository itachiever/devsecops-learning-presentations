The Tool
What is kubectl?
⌨️
Command-line tool for Kubernetes — your primary way to talk to a K8s cluster from the terminal
🔌
Sends requests to the API Server — every kubectl command is an HTTPS call to the API Server under the hood
📦
Manages all Kubernetes resources — Pods, Deployments, Services, ConfigMaps and much more
🌐
Works with any cluster — Minikube, Kubeadm, EKS, AKS, GKE — same commands everywhere
Pronunciation: "kube-control" or "kube-cuddle" — both are widely accepted 😄
How kubectl works
🧑‍💻
You (User)
Type kubectl command
↓ HTTPS request ↓
⌨️
kubectl CLI
Formats & sends request
↓ REST API call ↓
🔌
API Server
Validates & processes
↓ acts on cluster ↓
☸
K8s Cluster
Executes the action
↓ response ↓
pod/web-app created ✓

Syntax
kubectl Command Structure
Command Format
kubectl <action> <resource> [name] [flags]
kubectl
The CLI tool itself
<action>
get · apply · delete · describe · logs
<resource>
pods · nodes · services · deployments
[flags]
-n namespace · -o wide · --all
examples
# View pods
$ kubectl get pods
# View nodes
$ kubectl get nodes
# Apply config
$ kubectl apply -f app.yaml
# Delete pod
$ kubectl delete pod my-pod
# Describe pod
$ kubectl describe pod web
# View logs
$ kubectl logs web-pod

Viewing Resources
kubectl get & describe Commands
📋
kubectl get pods — lists all Pods in the current namespace with status
🖥
kubectl get nodes — lists all worker nodes and their Ready/NotReady status
🌐
kubectl get services — lists all Services and their types, IPs, and ports
🔎
kubectl describe pod <name> — full details: events, volumes, IP, image, restart count
🌍
kubectl get all — shows everything at once — pods, services, deployments, replicas

05 / 09
Creating & Managing
Create, Apply, Delete & Scale
✅
kubectl apply -f file.yaml — create or update resources from a YAML config file
🆕
kubectl create deployment <name> — quickly create a deployment from command line
🗑
kubectl delete pod <name> — remove a specific resource permanently
📈
kubectl scale --replicas=5 — scale a deployment to more or fewer Pods instantly
💡 Tip: Prefer kubectl apply over create — apply is idempotent (safe to run multiple times)
create & manage
# Apply from YAML (recommended)
$ kubectl apply -f deployment.yaml
deployment.apps/web created
# Quick create deployment
$ kubectl create deployment web --image=nginx
deployment.apps/web created
# Scale to 5 replicas
$ kubectl scale deployment web --replicas=5
deployment.apps/web scaled
# Delete a pod
$ kubectl delete pod web-abc123
pod "web-abc123" deleted

Troubleshooting
Debugging & Logs
📝
kubectl logs <pod-name> — view container logs — your app's stdout/stderr output
📝
kubectl logs -f <pod> — follow logs in real-time — like tail -f for your container
💻
kubectl exec -it <pod> -- bash — open a shell inside a running container
🔎
kubectl describe pod <name> — see events & errors — why is the pod crashing? Look here first.
🔧 Debug flow: get pods → describe → logs → exec — in this order to find most issues
debugging session
# Check what's wrong
$ kubectl get pods
web-pod    CrashLoopBackOff   0/1
# Read the logs
$ kubectl logs web-pod
Error: Cannot connect to DB
Connection refused on port 5432
# Look at events for more clues
$ kubectl describe pod web-pod
Events:
 Back-off restarting failed container
# Shell into a running container
$ kubectl exec -it web-pod -- bash
root@web-pod:/# █

Real-World
A Typical kubectl Workflow
1
Write your YAML manifest
apiVersion: apps/v1
kind: Deployment
metadata:
  name: my-app
↓
2
Apply the manifest
$ kubectl apply -f my-app.yaml
↓
3
Check status
$ kubectl get pods && kubectl get svc
↓
4
Debug if issues
$ kubectl logs my-app-abc && kubectl describe pod

5
Update & re-apply
$ kubectl apply -f my-app.yaml # repeat as needed
