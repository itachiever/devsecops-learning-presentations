What is Docker?

🚢 Think of Docker like Shipping Containers

Just like shipping containers revolutionized global trade, Docker containers revolutionize software deployment!

📦

Physical Container

Standardized box that holds goods safely during transport

🐳

Docker Container

Standardized package that holds your app and everything it needs

Docker is a platform that packages your application with all its dependencies into lightweight, portable containers.

3 / 6

Core Components

Docker has three main parts that work together like a team:

👤

Docker Client

The Command Center
Where you type commands like "docker run". It's like the steering wheel of a car - you use it to control Docker.

⚙️

Docker Daemon

The Engine
The powerful engine running in the background. It does all the heavy work - building, running, and managing containers.

🏪

Docker Registry

The App Store
Like an app store where you can download pre-built images (Docker Hub) or store your own custom images.

Docker Images & Containers

🍳 Think of it like Cooking!

Docker Image = Recipe | Docker Container = The Finished Dish

📋

Docker Image

The Recipe
Blueprint with instructions and ingredients. Read-only template that doesn't change.

➡️
🍽️

Docker Container

The Finished Dish
Running instance of the image. You can have multiple dishes from one recipe!

🔄 One Image, Many Containers

nginx:latest Image → Container 1 (Port 80) + Container 2 (Port 8080) + Container 3 (Port 9000)
