🖼️ Container Images & Vulnerabilities

Bad Images = Security Nightmares

🏗️

Image Problems

  • Untrusted sources (random internet images)
  • Hidden malware or backdoors
  • Bloated images with unnecessary tools
  • Use official images from Docker Hub
📅

Outdated Software

  • Old versions have known security holes
  • Attackers target these weaknesses
  • Scan images before using them
  • Update regularly
  • Use minimal base images
🔍 Always scan your images and keep them updated!
Risk Level:
HIGH

🏠 Docker Host & Container Breakouts

When Containers Escape Their Cage

🖥️

Insecure Docker Host

  • Unpatched host operating system
  • Docker daemon exposed to internet
  • Weak access controls
  • Keep host OS updated
  • Secure Docker daemon
🏃‍♂️

Container Breakouts

  • Containers running as root user
  • Privileged mode enabled
  • Shared kernel vulnerabilities
  • Use non-root users
  • Avoid privileged mode

🔐 Secrets & Default Security Settings

Don't Leave the Door Wide Open

🗝️

Secrets Management

  • Passwords hardcoded in images
  • API keys in environment variables
  • Database credentials exposed
  • Use Docker secrets or vault systems
  • Never put secrets in code
⚙️

Default Security Settings

  • Default passwords unchanged
  • Unnecessary ports exposed
  • Weak security configurations
  • Change all defaults
  • Follow principle of least privilege
🚨 "admin:password" is NOT a good secret!

Treat secrets like your wallet - keep them safe and out of sight!

🌐 Insecure Networking & Volumes

Network Holes & Data Leaks

📡

Insecure Networking

  • All ports exposed to internet
  • No network segmentation
  • Unencrypted communications
  • Use custom networks
  • Expose only necessary ports
💾

Insecure Volumes

  • Mounting sensitive host directories
  • Write access to host filesystem
  • Shared volumes between containers
  • Use read-only mounts when possible
  • Limit volume access

💻 Resource Usage Abuse

When Containers Become Resource Hogs

🐷

Resource Hogging

  • No CPU or memory limits set
  • One container crashes the host
  • DoS attacks through resource exhaustion
  • Set resource limits always
  • Monitor resource usage
📊

Common Problems

  • Memory leaks in applications
  • Infinite loops consuming CPU
  • Fork bombs creating processes
  • Use Docker resource constraints
  • Implement health checks
⚡ Set limits: --memory=512m --cpus=1.0
